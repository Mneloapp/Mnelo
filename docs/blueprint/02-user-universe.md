# User Universe

## Primary Users

Mnelo is built for construction teams that turn project documents into estimates, RFQs, supplier comparisons and procurement decisions.

| User | Core Need | Mnelo Value |
| --- | --- | --- |
| Estimator | Understand scope and prepare pricing | Structured BOQ extraction, classification and reusable knowledge |
| Procurement Manager | Source suppliers and manage RFQs | RFQ package generation, supplier matching and quote tracking |
| Project Manager | Understand project cost and procurement status | Connected project intelligence and decision visibility |
| Quantity Surveyor | Validate quantities and commercial structure | Explainable takeoff, review queues and version history |
| Commercial Manager | Control cost, risk and margin | Quote intelligence, estimate history and audit trail |
| Company Owner | Improve speed, consistency and profitability | Knowledge reuse and reduced manual coordination |

## Initial Market

Mnelo starts with MEP contractors because MEP procurement depends on large BOQs, complex technical specifications and many specialized suppliers.

The product must not become MEP-only. All terminology, workflows and data structures must support other disciplines:

| Discipline | Example Packages |
| --- | --- |
| MEP | HVAC, plumbing, fire fighting, electrical, low current |
| Civil | earthworks, drainage, external works, roads |
| Structural | concrete, reinforcement, steel, embeds |
| Architectural | finishes, doors, windows, facades |
| Industrial | equipment, piping, instrumentation, automation |
| Fit-out | furniture, lighting, partitions, millwork |

## User Jobs

| Job | Current Pain | Desired Mnelo Outcome |
| --- | --- | --- |
| Upload tender package | Files are scattered and manually reviewed | Documents are classified and routed automatically |
| Parse BOQ | Excel structure is inconsistent | Rows, quantities, units and sections are extracted reliably |
| Classify items | Manual category assignment is slow | Products are grouped by system, category and subcategory |
| Prepare RFQs | Packages are manually copied into spreadsheets | RFQs are generated from verified knowledge |
| Compare quotes | Supplier responses arrive in different formats | Quotes are normalized and compared against the same scope |
| Reuse decisions | Prior knowledge is lost in old files | Corrections and decisions improve future projects |

## Trust Requirements

Construction users will not trust a black box. Mnelo must show:

- What the AI changed.
- Why it made the decision.
- What confidence score was assigned.
- What source document or previous correction influenced the result.
- Whether a human confirmed or corrected the result.
