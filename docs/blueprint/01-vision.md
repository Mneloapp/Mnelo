# Mnelo Vision

## Purpose

Mnelo is an AI Procurement Operating System for Construction. It turns fragmented project documents, BOQs, supplier information, quotes and procurement activity into structured, explainable and reusable knowledge.

Mnelo starts with MEP contractors because their estimation and procurement workflows are document-heavy, revision-heavy and supplier-dependent. The platform is designed from the beginning to become multi-discipline across construction, infrastructure, industrial, fit-out and specialty contractor workflows.

## Product Thesis

Construction procurement is not only a purchasing workflow. It is a knowledge workflow.

Every tender package contains valuable knowledge:

| Knowledge Area | Examples |
| --- | --- |
| Project scope | Systems, packages, specifications, drawings, BOQ sections |
| Commercial logic | Quantities, units, rates, alternates, exclusions |
| Supplier intelligence | Who can supply what, lead times, pricing history, responsiveness |
| Decision history | What was reviewed, changed, approved, rejected or clarified |
| Reusable patterns | Product mappings, category decisions, package structures, supplier matches |

Mnelo’s core job is to capture this knowledge once and reuse it across future estimates, RFQs, quotes and procurement decisions.

## Principles

1. Mnelo is AI-native, not AI-assisted after the fact.
2. AI must always explain its decisions.
3. Every feature must save time, save money or create reusable procurement knowledge.
4. Knowledge is more valuable than software.
5. The product must start simple, but its data model must support long-term intelligence.
6. Construction teams must always be able to review, correct and teach the system.

## Target Operating Model

Mnelo transforms the procurement journey into a connected workflow:

```text
Project
→ Documents
→ BOQ Parsing
→ Entity Extraction
→ Classification
→ Knowledge Graph
→ Systems
→ RFQ Packages
→ Suppliers
→ Quotes
→ Estimate
→ Procurement
```

## Long-Term Outcome

Mnelo becomes the operating system that helps construction teams understand what needs to be bought, who can supply it, what it should cost, what changed and why each procurement decision was made.
